# Accessibility — Form Builder

> Status: baseline audit on 2026-06-03. Manual checks against the deployed `forms.manne.work` iframe.

## Keyboard Navigation

| Shortcut | Action | Where |
|---|---|---|
| `Tab` / `Shift+Tab` | Move focus | All interactive controls |
| `Enter` / `Space` | Activate | Buttons, palette items |
| `Delete` / `Backspace` | Delete selected field | Canvas (focus outside inputs) |
| `Cmd/Ctrl+S` | Save | Global |
| `Cmd/Ctrl+Z` | Undo | Global |
| `Shift+Cmd/Ctrl+Z` | Redo | Global |
| `ArrowUp` / `ArrowDown` | Reorder selected | Canvas |
| `Esc` | Close dialog | Delete confirmation |

**Input guard:** keyboard shortcuts ignore events whose `target` is `INPUT`/`TEXTAREA`/`SELECT`, so typing is never hijacked.

## Focus indicators

- All focusable elements get a 2px ring outline via `:focus-visible` in `app/globals.css`.
- The browser's default focus ring is suppressed for non-keyboard activation (mouse clicks) to keep the visual canvas clean.

| Element | Roles / Properties | Source |
|---|---|---|

| Drag handle | `aria-label="Drag to reorder"` | `src/components/ui/field-card.tsx:43` |

## Live regions

- **Canvas reorder** — `aria-live="polite"` on the canvas wrapper announces field position changes (move, delete, add) without interrupting the user.
- **Save success** — Sonner toast (added in PR 5) — uses `role="status"` semantics; non-blocking.

## Form labels

- All entity components in the canvas render the label as `<Label>` (shadcn), with the `*` required indicator paired via `htmlFor`/`id` linking. Rendered labels and inputs are uniquely id'd per entity using the entity id.

## Color contrast

- The shadcn taupe palette (`--background`, `--foreground`, `--primary`, `--muted-foreground`) targets WCAG AA contrast for text on background.
- The destructive ring is `oklch(0.577 0.245 27.325)` — verified against `--background` to clear AA on large text. Small text uses `--destructive-foreground` for safety.

## What's NOT covered (future work)

- [ ] High-contrast / forced-colors mode (`@media (forced-colors)`) — relies on OS settings today.
- [ ] Screen reader announcement of field type on add/select (currently announced via live region but as a generic "field added").
- [ ] Sortable item position announcements (`aria-posinset`/`aria-setsize`) — dnd-kit has it; we don't pipe it through yet.
- [ ] Reduced-motion preference on the toast slide-in animation.

## Manual test plan

1. Load `https://forms.manne.work/public/test-host.html`.
2. `Tab` from the URL bar: focus should land on the first palette item with a visible ring.
3. `Tab` through all palette items → into canvas → into properties panel — order is logical.
4. Press `Delete` on a selected field — focus returns to the next field (or to empty state).
5. `Cmd+S` — toast appears, `aria-live` announces "Form saved".
6. From a different origin, post a `LOAD_FORM` — toast appears with rejection message.
